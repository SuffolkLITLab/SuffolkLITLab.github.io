How To...

Image and Hover Text: Tech Support Cheat Sheet by XKCD

This XKCD comic is a pretty good primer on "learning" your way around new software. However, we saw a benefit in providing more formal materials, hence LIT HOW-TO. The following documentation is used to help bring lab members up to speed on a constellation of tech tools and concepts used in our work. They're a work in progress, but we hope even in their current state others might find them useful too. So please, poke around.

One could start at the beginning and work their way through all of the "lessons," however, it's more common for us to reference sections as projects call for them. That being said, the first section is a good place to start. We plan to use these resources heavily in next Fall's Coding the Law and as a supplement for LIT Fellows and students in the LIT Lab.

Self-direction is the name of the game. However, students, and to a lessor extent the general public, are not playing without a net. All are welcome to join the Lab's Slack Team. There you'll find public channels for most of the sections below. We hope you'll use them to ask and answer questions.

The Lab's standing rule re. asking for help is as follows: If you set to doing something and you are hitting your head against a wall for 30 plus minutes, ask for help in the appropriate Slack channel.

If you're not a Lab student and you are using one of the channels associated with a section below, you must abide by the following three rules:

  1. You've got to be kind;†
  2. If you're asking a question, don't expect an immediate answer (remember Rule 1);
  3. If you're answering a question (which you're encouraged to do), don't provide an answer out of the gate. Work to point people in the right direction; and
  4. Fill out your user profile so we know who you are.

† Lab faculty are ultimate arbiters for all disputes.

3. Share Your Work on Github

Git is a system for managing the many versions of code one writes over the course of a coding project. At one level, it can act like track changes for computer code, allowing multiple people to collaborate on a single project. GitHub is a website on which you can host and manage projects using Git. It's also a community of users where people share their code.     6. Build Simple Expert Systems ("Chatbots")

    Most of the time when you interact with a "chatbot," you're interacting with a glorified decision tree. QnA is a markup language for people with little or no programming experience. Designed with attorneys in mind, it transforms blocks of text into interactive question and answer sessions (QnA interviews). Such rules-based systems are sometimes called expert systems, and the idea is that one can encode some small area of knowledge as part of an interactive decision tree, making it more broadly available. In the following lessons you will learn how to turn flowcharts into functioning QnA interviews/"chatbots."

  • QnA Markup (~?m)
  • Crafting a Great Flowchart (Case Study) (~?m)
  • Publish a QnA Interview (~?m)
  • QnA + JavaScript (~10m)
  • Challenge: Do You Qualify (~10m)
